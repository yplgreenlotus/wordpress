# 概述

## 1、面向过程编程

为了证明上面概述所说的简单话，我可以用面向过程编程的思想对比下：

**例如：吃饭**

面向过程的做法如下：

**煮饭、买菜、洗菜、切菜、炒菜、盛菜、吃饭**

面向对象的做法如下：

**做饭、吃饭**

对比一下我们就能理解，面向过程与面向对象的区别，面向过程它注重整个煮饭、炒菜的过程，而面向对象则是一个做饭菜将整个过程抽象到了里面，符合将复杂的事情简单化，不关心性具体的过程，是不是跟贴切人类思考方向

## 2、Class（类）

在面向对象编程的世界里面一切从繁化简，说起起来是不是特么的容易，但是话说没有规矩不成方圆，那么这个规矩就是类。

大家都知道 在编程的过程中 对象 是在 类 的基础之上创建出来的，如果你想让 对象实例 具备那些行为特征 ，是不是先要在类里面定义相关的属性行为特征，说到这里就很明显了，类是用来 声明、收集、定义等 具体的行为特征模板，根据这个事先预定义好的模板创建相应的实例对象。

**总结：定义对象的行为特征。它是对象的属性和方法的模板定义。**

## 3、Constructor\(构造函数\)

对象初始化的瞬间，被调用的方法。通常它的名字与包含它的类一致。一般情况下构造函数如果不显示声明，都是默认存在

## 4、Encapsulation （封装）

封装是面向对象的三大特征之一，它就是把客观事物的属性、行为、特征等抽象到一个类里面，类可以定义这些属性、行为、特征等的可访问程度（private、protected、public）等，以防止程序中无关的部分意外的改变或错误的使用了对象的私有部分

## 5、Inheritance （继承）

继承是面向对象的三大特征之一，它就是把客观事物的属性、行为、特征等从一个类里面承接过渡的另一个类里面，这样也就有了 父类（基类、超类）、子类之说。

**继承概念的实现方式有二类：**

**1、** **实现继承：**是指直接使用基类的属性和方法而无需额外编码的能力

**2、接口继承：**是指仅使用属性和方法的名称、但是子类必须提供实现的能力

## 6、Polymorphism （多态）

多态是面向对象的三大特征之一，从字面意思理解就是多种形态，比如 人 有 黄、黑、白 3 种 肤色，其实就是多种形态的表现等，如果运用在编程领域 就是 一个实例对象里面的相同方法在不同情形下有不同的表现形式，最直观的就是 重写 与 重载。

**重写 ：**子类继承父类的方法，但是由于子类可能有不同的形态，所以子类重写了父类继承而来的方法。方法重写又称方法覆盖。

**重载：** 实例对象内同名不同参数的函数或者方法之间，互相称之为重载函数或者方法
