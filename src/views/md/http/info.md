# 如何理解HTTP？

> #### HTTP 是什么 ?

如果有人问你HTTP是什么? 你回答 HTTP是超文本传输协议。其实这样回答确实是没有什么问题，但是也不可否认这样回答的太过于片面。要想理解的更加深入，我们得从 “HTTP是超文本传输协议” 这句话中提取3个关键词，分别是：“超文本”、“传输”、“协议”。

> #### 如何理解 “协议” ?

举个例子：
入职新的公司，公司要跟你签定入职合同，技术保密协议等，里面都包含了各种各样的规则，如：薪水、试用期时间、合同到期时间、公司上下班时间、泄露公司机密的处罚措施等等。那么从中可以看出协议是建立在两者及以上的参与者之上的，同时协议也是双向的，协议里面规定了两者之间的 “责、权、利”。

如果把这些规定放在计算机之间的通信上，那么 HTTP 是一个用在计算机世界里面的协议，它使用计算机能够理解的语言确立了一种计算机之间交流通信的规范，以及相关的各种控制和错误的处理方式

> #### 如何理解 “传输” ?

所谓的传输就是从 A 点传到 B 点，或者从 B 点传到 A 点，但是从 A 到 B 的过程中并没有限制经过了多少个“中转”，可能在传输的过程中经过了 X、Y、Z、...个中转最后在到达 B 点，那么这些中转可能是：安全认证、数据压缩、编码转换等一系列的处理，优化传输过程。
那么如果从传输层面来理解， http 是一个在计算机世界里专门用来在两点之间传输数据的约定和规范


> #### 如何理解 “超文本” ?

所谓的 “文本” 就是一些可以被 浏览器、服务器所理解的普通文本数据，而不是像 TCP/UDP 这样的底层协议所传输的数据二进制数据包。

所谓的 “超文本” 就是 “超越了普通文本的文本”，最熟悉的就是 html了，它包含了文字、图片、音频、视频、超链接等。也正因为超链接，可以从一个文本跳转到另一个文本从而生成了复杂的非线性、网状结构。

从超文本这边个层面来理解，那么http 是一个在计算机世界里面专门传输文字，图片、音频、视频等超文本数据的约定和规范
